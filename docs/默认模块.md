# ODM接口/任务管理

<a id="opIdtask_new_post"></a>

## POST 创建新任务

POST /task/new

创建新任务并将其置于处理队列末尾。对于上传非常大的任务，请改用 /task/new/init

> Body 请求参数

```yaml
name: ""
images: ""
options: ""
outputs: ""
skipPostProcessing: ""
webhook: ""
zipurl: ""
dateCreated: 0

```

### 请求参数

|名称|位置|类型|必选|说明|
|---|---|---|---|---|
|token|query|string| 否 |认证所需的令牌（当需要认证时）|
|set-uuid|header|string| 否 |可选的 UUID 字符串，将用作此任务的 UUID 而不是生成随机 UUID|
|body|body|object| 否 |none|
|» name|body|string| 否 |要与任务关联的可选名称|
|» images|body|string(binary)| 否 |要处理的图像，以及可选文件如 GEO 文件 (geo.txt)、图像组文件 (image_groups.txt)、GCP 文件 (*.txt)、种子文件 (seed.zip) 或对齐文件 (align.las, align.laz, align.tif)。如果包含 GCP 文件，应使用 .txt 扩展名。如果包含种子归档，则用其内容预填充任务目录|
|» options|body|string| 否 |用于处理的选项的序列化 JSON 字符串，格式为数组：[{name: option1, value: value1}, {name: option2, value: value2}, ...]。例如：[{"name":"cmvs-maxImages","value":"500"},{"name":"time","value":true}]。调用 /options 可获取所有选项列表|
|» outputs|body|string| 否 |可选的序列化 JSON 字符串，指定相对于项目目录的路径，这些路径应包含在 all.zip 结果文件中，覆盖默认行为|
|» skipPostProcessing|body|boolean| 否 |设置后，跳过点云瓦片的生成|
|» webhook|body|string| 否 |处理结束时（无论成功或失败）调用的可选 URL|
|» zipurl|body|string| 否 |包含要处理的图像的 zip 文件的 URL，以及可选的 GEO 文件和/或 GCP 文件。如果包含 GCP 文件，应使用 .txt 扩展名|
|» dateCreated|body|integer| 否 |可选的时间戳，覆盖任务的默认创建日期|

> 返回示例

> 200 Response

```json
{
  "uuid": "string"
}
```

### 返回结果

|状态码|状态码含义|说明|数据模型|
|---|---|---|---|
|200|[OK](https://tools.ietf.org/html/rfc7231#section-6.3.1)|成功|[TaskNewResponse](#schematasknewresponse)|
|default|Default|错误|[Error](#schemaerror)|

<a id="schema_NodeInfo"></a>
<a id="tocSnodeinfo"></a>
<a id="tocsnodeinfo"></a>

```json
{
  "availableMemory": 0,
  "cpuCores": 0,
  "engine": "string",
  "engineVersion": "string",
  "maxImages": 0,
  "maxParallelTasks": 0,
  "taskQueueCount": 0,
  "totalMemory": 0,
  "version": "string"
}

```

节点信息

### 属性

|名称|类型|必选|约束|中文名|说明|
|---|---|---|---|---|---|
|availableMemory|integer|false|none||可用 RAM 量（字节）|
|cpuCores|integer|false|none||CPU 核心数（虚拟）|
|engine|string|true|none||处理引擎的小写标识符|
|engineVersion|string|true|none||处理引擎的当前版本|
|maxImages|integer|true|none||新任务允许的最大图像数，如果无限制则为 null|
|maxParallelTasks|integer|false|none||可以同时处理的最大任务数|
|taskQueueCount|integer|true|none||当前正在处理或等待处理的任务数|
|totalMemory|integer|false|none||系统中的总 RAM 量（字节）|
|version|string|true|none||当前 API 版本|

<a id="schemaoption"></a>
<a id="schema_Option"></a>

<a id="tocsoption"></a>

```json
{
  "domain": "string",
  "help": "string",
  "name": "string",
  "type": "int",
  "value": "string"
}

```

处理选项

### 属性

|名称|类型|必选|约束|中文名|说明|
|---|---|---|---|---|---|
|domain|string|true|none||有效值范围（例如 "正整数" 或 "浮点数 > 0.0"）|
|help|string|true|none||此选项的说明|
|name|string|true|none||命令行选项（与传递给 OpenDroneMap 进程的完全相同，不包括前导 '--'）|
|type|string|true|none||此选项值的数据类型|
|value|string|true|none||此选项的默认值|

#### 枚举值

|属性|值|
|---|---|
|type|int|
|type|float|
|type|string|
|type|bool|

<h2 id="tocS_TaskInfo">TaskInfo</h2>

<a id="schemataskinfo"></a>
<a id="schema_TaskInfo"></a>
<a id="tocStaskinfo"></a>
<a id="tocstaskinfo"></a>

```json
{
  "dateCreated": 0,
  "imagesCount": 0,
  "name": "string",
  "options": [
    {
      "name": "string",
      "value": "string"
    }
  ],
  "output": [
    "string"
  ],
  "processingTime": 0,
  "status": {
    "code": 0
  },
  "uuid": "string",
  "progress": 0
}

```

任务详细信息

### 属性

|名称|类型|必选|约束|中文名|说明|
|---|---|---|---|---|---|
|dateCreated|integer|true|none||时间戳|
|imagesCount|integer|true|none||图像数量|
|name|string|true|none||名称|
|options|[[TaskOption](#schemataskoption)]|true|none||用于处理此任务的选项列表|
|output|[string]|false|none||任务的控制台输出（仅当通过 ?output=<linenum> 请求时）|
|processingTime|integer|true|none||自任务开始处理以来经过的毫秒数|
|status|[TaskStatus](#schemataskstatus)|true|none||任务状态|
|uuid|string|true|none||UUID|
|progress|integer|true|none||此任务的进度，从 0 到 100|
